Puppet Defined Type: vip

Defined in:
manifests/vip.pp

Overview

Configure a Virtual IP address configured with VRRP

This resource relies on keepalived.

Parameter Summary

  • net_id (Any) -

    Network ID (wan, administration...) of the network where this VIP will be.

  • ip_address (Any) -

    IPv4 address of VIP

  • router_id (Any) -

    Unique integer for this VIP, it should be unique on the the L2 segment. Cluster sharing an external network should not use the same ids.

  • auth_secret (Any) -

    Secret shared between the routers to authenticate the origin.

  • master (Any) -

    Is this node the master for this vip

  • priority (Any)
  • prefix (Any) => '' -

    Prefix for the name of the instance

  • notify_script (Any) => false -

    Configure a notify script (default: false)

  • advert_int (Any) => undef -

    Interval between advertisement (heartbeat) (default: 1s)

  • prority (Unknown) -

    Integer giving the priority of this node for this vip
